# 🖼️ AI Generated Image Detector

> Detecting whether an image was photographed from real life or generated by an AI model — trained on **120,000 [CIFAKE](https://www.kaggle.com/datasets/birdy654/cifake-real-and-ai-generated-synthetic-images) images** (60K real CIFAR-10 + 60K Stable Diffusion generated).

Fine-tuned ResNet-18 achieves **98.42% accuracy** while CLIP zero-shot struggles at 55% — demonstrating that detecting AI-generated images requires task-specific training, not just general visual understanding.

The best model is served as a **production REST API** via FastAPI + Docker.

---

## 📦 Dataset

[CIFAKE](https://www.kaggle.com/datasets/birdy654/cifake-real-and-ai-generated-synthetic-images) — 120,000 images across two classes:
- **REAL**: 60,000 images from CIFAR-10 (real photographs)
- **FAKE**: 60,000 images generated by Stable Diffusion

Train: 100,000 | Test: 20,000 | Balanced classes

---

## 🎯 Problem

AI-generated synthetic images are increasingly indistinguishable from real photographs, posing risks across misinformation, fraud, and content integrity. This project builds a binary classifier to detect whether an image is real or AI-generated, comparing a fine-tuned CNN against a zero-shot vision-language model (CLIP) to understand what approach works and why.

---

## 📊 Results

| Model | Accuracy | AUC-ROC | Macro F1 | Type |
|-------|----------|---------|----------|------|
| **ResNet-18 (fine-tuned)** | **98.42%** | **0.9986** | **0.9842** | Supervised |
| CLIP | 55.49% | 0.6826 | 0.4582 | Zero-shot |

### 🔑 Key Finding

CLIP — despite being a powerful vision-language model trained on 400M image-text pairs — achieves only 55.49% accuracy on this task, barely above random. ResNet-18 fine-tuned on domain-specific data achieves 98.42%. This 43-point gap confirms that **AI-generated image detection is a domain-specific problem** that general-purpose zero-shot models cannot solve without task-specific adaptation.

## 🔍 Approach

### 1. Exploratory Data Analysis
- Visualized sample real vs AI-generated images across CIFAR-10 categories
- Computed pixel intensity statistics — real images (mean=0.470, std=0.210) vs AI-generated (mean=0.425, std=0.222), showing subtle but consistent distributional differences

### 2. Model 1 — ResNet-18 (CNN Baseline, Fine-tuned)
- Loaded ImageNet pretrained weights
- Froze early layers, fine-tuned last 2 residual blocks + classifier head
- Added Dropout(0.3) before classifier to reduce overfitting
- Trained with AdamW + CosineAnnealingLR for 10 epochs
- Data augmentation: horizontal flip, rotation, color jitter

### 3. Model 2 — CLIP Zero-Shot (No Training)
- Used `openai/clip-vit-base-patch32` with 3 prompt pair strategies
- Best prompt: *"a real photograph"* vs *"an AI-generated image"* — AUC 0.6636
- No fine-tuning — evaluates CLIP's out-of-the-box capability on this task

### 4. Evaluation
- Accuracy, AUC-ROC, Macro F1, confusion matrix, ROC curves
- Misclassified example analysis to understand failure modes

## 🚀 How to Run

### Option 1 — Docker (recommended)
```bash
docker-compose up --build
```

Send a prediction:
```bash
curl -X POST http://localhost:8000/predict \
  -F "file=@image.jpg"
```

Expected response:
```json
{
  "label": "FAKE",
  "confidence": 0.97,
  "probabilities": {"FAKE": 0.97, "REAL": 0.03},
  "latency_ms": 18.4
}
```

### Option 2 — Local API
```bash
pip install -r requirements.txt
uvicorn src.api:app --reload --port 8000
```

### Option 3 — Train from Scratch (Colab)
Upload `cifake.zip` and run `notebooks/cifake_detection.ipynb`.
Enable GPU: Runtime → Change runtime type → T4 GPU.
